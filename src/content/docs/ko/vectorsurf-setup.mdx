## 사전 요구사항

VectorSurfer를 사용하기 전에 필요한 것:

- Python 프로젝트에 **VectorWave**가 설치 및 설정되어 있어야 합니다
- VectorWave 데이터가 있는 **Weaviate** 인스턴스가 실행 중이어야 합니다
- 최신 **웹 브라우저**

## 1단계: VectorWave 설정

아직 설정하지 않았다면, VectorWave를 설치하고 함수 실행 로깅을 시작하세요:

```bash
pip install vectorwave
```

```python
from vectorwave import vectorize, initialize_database

initialize_database()

@vectorize(auto=True, capture_return_value=True)
def your_function(prompt: str):
    # AI 함수 로직
    return result
```

`@vectorize`로 데코레이팅된 함수는 모든 실행이 자동으로 Weaviate에 저장됩니다.

자세한 설정 방법은 [VectorWave 빠른 시작](/docs/quickstart)을 참고하세요.

## 2단계: 대시보드 열기

VectorSurfer 대시보드를 방문하세요:

**[대시보드 열기 →](https://test3-six-rose.vercel.app)**

VectorSurfer는 VectorWave가 데이터를 쓰는 동일한 Weaviate 인스턴스에서 읽습니다. 연결되면 모든 함수 데이터가 자동으로 표시됩니다.

## 3단계: 데이터 탐색

VectorWave가 실행을 기록하면 대시보드에 다음이 표시됩니다:

- 실시간 지표가 포함된 **KPI 카드**
- 시간별 함수 활동을 보여주는 **실행 타임라인**
- 필터링을 위한 사이드바의 **함수 목록**

대시보드에 데이터가 표시되지 않으면, VectorWave로 최소 하나의 함수 실행이 기록되었는지 확인하세요.

## Weaviate 연결

VectorSurfer는 VectorWave에 설정된 Weaviate 인스턴스에 연결합니다. Weaviate 인스턴스가 대시보드에서 접근 가능한지 확인하세요.

로컬 개발 시 Weaviate는 보통 `http://localhost:8080`에서 실행됩니다. 프로덕션 환경에서는 클라우드 Weaviate 엔드포인트를 사용하세요.

## 선택사항: AI Healer 설정

AI 기반 에러 진단과 자동 GitHub PR 기능을 활성화하려면:

- 대시보드 설정에서 **OpenAI API 키**를 구성하세요
- 자동 PR 생성을 위해 **GitHub 리포지토리**를 연결하세요

이 기능들은 선택사항입니다 — 대시보드는 이 기능 없이도 모니터링과 트레이싱이 가능합니다.
