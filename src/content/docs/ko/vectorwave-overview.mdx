## VectorWave란?

**VectorWave**는 LLM 통합 애플리케이션에서 "효율성 vs 신뢰성" 딜레마를 해결하기 위해 설계된 통합 프레임워크입니다. **실행 레벨 시맨틱 최적화**와 **자율 자동 치유**를 결합합니다.

코드가 실행되는 순간 사라지는 휘발성 데이터를 **검색 가능하고 재사용 가능한 영구 지식 자산**으로 변환합니다.

기존 시맨틱 캐싱 도구가 텍스트 유사도에 집중하는 것과 달리, VectorWave는 전체 **함수 실행 컨텍스트**를 캡처합니다. 성공적인 실행으로부터 영구 "Golden Dataset"을 생성하여:

1. **비용 절감** — 시맨틱으로 유사한 입력에 캐시된 결과를 제공, 비싼 연산을 건너뜁니다.
2. **버그 수정** — 런타임 에러를 자동 진단하고 LLM으로 GitHub PR을 생성합니다.
3. **품질 모니터링** — 사용자 입력이 알려진 패턴에서 벗어나기 시작할 때 감지합니다 (시맨틱 드리프트).

## 아키텍처

VectorWave의 아키텍처는 세 가지 핵심 컴포넌트로 구성됩니다:

```
┌──────────────────────────────────────────────────┐
│                   VectorWave                      │
│                                                   │
│  ┌─────────────────┐  ┌──────────────────────┐   │
│  │ 최적화           │  │ 트레이스 컨텍스트      │   │
│  │ 엔진            │  │ 매니저                │   │
│  │                  │  │                       │   │
│  │ • 캐시 히트 검사  │  │ • 실행 로깅           │   │
│  │ • HNSW 인덱스    │  │ • 입출력 저장          │   │
│  │ • 배치 라이트    │  │ • trace_id / span_id  │   │
│  └────────┬─────────┘  └───────────┬───────────┘   │
│           │                        │               │
│  ┌────────┴────────────────────────┴───────────┐   │
│  │           자동 치유 파이프라인                 │   │
│  │                                              │   │
│  │  • 에러 감지 → RAG 진단                       │   │
│  │  • LLM 코드 수정 → AST 패칭                  │   │
│  │  • 브랜치 생성 → GitHub PR                    │   │
│  └──────────────────────────────────────────────┘   │
│                        │                            │
│                    Weaviate                          │
│              (벡터 데이터베이스)                       │
└──────────────────────────────────────────────────────┘
```

### 핵심 컴포넌트

- **최적화 엔진** — 함수 호출을 인터셉트하여 HNSW 인덱스로 시맨틱 캐시 히트를 확인합니다. 히트 시 함수를 재실행하는 대신 ~0.02초 만에 캐시된 결과를 반환합니다.
- **트레이스 컨텍스트 매니저** — 코드 구조를 수정하지 않고 실행 로그, 입력, 출력을 수집합니다. `trace_id`와 `span_id`로 계층적 분산 트레이싱을 지원합니다.
- **자동 치유 파이프라인** — 에러 발생 시 깨어나 RAG로 근본 원인을 진단하고, AST 파싱으로 코드 수정을 생성하며, GitHub Pull Request를 제출하는 자율 에이전트입니다.

### 데이터 저장

VectorWave는 네 개의 Weaviate 컬렉션에 데이터를 저장합니다:

| 컬렉션 | 목적 | 데이터 |
|---|---|---|
| `VectorWaveFunctions` | 정적 함수 메타데이터 | 소스 코드, 독스트링, 설명, 모듈 경로 |
| `VectorWaveExecutions` | 동적 실행 로그 | 입력, 출력, 에러, 트레이스 ID, 타임스탬프, 임베딩 |
| `VectorWaveGoldenDataset` | 검증된 고품질 실행 | 캐시 우선순위 및 드리프트 기준선을 위한 큐레이션된 결과 |
| `VectorWaveTokenUsage` | LLM 토큰 추적 | 모델, 사용 유형, 토큰 수 (비용 모니터링용) |

> **VectorSurfer**: 모든 컬렉션 데이터는 [VectorSurfer 대시보드](/vectorsurf)에서 시각화됩니다 — KPI 카드, 실행 타임라인, 트레이스 워터폴, Golden Dataset 관리.

## 성능 벤치마크

| 지표 | 직접 실행 | VectorWave 적용 | 개선율 |
|---|---|---|---|
| **레이턴시 (히트)** | ~2.5초 (LLM API) | **~0.02초** | **125배 빠름** |
| **비용 (히트)** | $0.03 / 호출 | **$0.00** | **100% 절감** |
| **신뢰성** | 수동 수정 필요 | **자동 PR 생성** | **자율화** |

## 기능 비교

| 기능 | 기존 도구 (예: GPTCache) | **VectorWave** |
|---|---|---|
| **시맨틱 캐싱** | 텍스트 기반 | **실행 컨텍스트 기반** |
| **자동 치유 (자동 수정)** | 없음 | **자율 동작** |
| **GitOps (자동 PR)** | 없음 | **원활한 통합** |
| **시맨틱 드리프트 감지** | 없음 | **드리프트 레이더** |
| **제로 설정** | 설정 필요 | **데코레이터 하나** |

## 생태계

VectorWave는 cozymori 생태계의 코어 SDK입니다:

```
┌──────────────────────────────────────────┐
│              VectorWave                   │
│  (코어 SDK — 추적, 캐싱, 치유)            │
│  실행 데이터를 Weaviate에 저장             │
└──────────┬───────────┬───────────┬───────┘
           │           │           │
           v           v           v
┌──────────┐ ┌────────────┐ ┌───────────┐
│VectorSurfer│ │VectorSurfer│ │VectorCheck│
│(Next.js +  │ │STL         │ │(CLI 테스트 │
│ FastAPI)   │ │(Streamlit) │ │ 프레임워크)│
└────────────┘ └────────────┘ └───────────┘
```

- **[VectorSurfer](/docs/vectorsurf-overview)** — AI 관측을 위한 풀스택 웹 대시보드
- **[VectorSurferSTL](/docs/vectorsurferstl)** — 경량 Streamlit 대시보드
- **[VectorCheck](/docs/vectorcheck)** — CLI 회귀 테스트 프레임워크
